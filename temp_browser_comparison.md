# 轻量化浏览器对比分析

## 一、浏览器选项对比

| 浏览器 | 类型 | 大小 | JavaScript | 用户交互 | GitHub 兼容 | 推荐场景 |
|--------|------|------|------------|----------|-------------|----------|
| **Playwright/Chromium** | 完整浏览器 | ~200MB | ✅ 完整 | ✅ 支持 | ✅ 完全兼容 | Token 获取 |
| **Lightpanda** | 轻量无头 | ~10MB | ✅ V8 | ❌ 无头 | ⚠️ Beta | 爬虫/测试 |
| **Puppeteer** | 完整浏览器 | ~200MB | ✅ 完整 | ✅ 支持 | ✅ 完全兼容 | Token 获取 |
| **Selenium** | 浏览器驱动 | 依赖浏览器 | ✅ 完整 | ✅ 支持 | ✅ 完全兼容 | 通用自动化 |
| **requests-html** | 轻量 | ~5MB | ⚠️ 有限 | ❌ 无 | ❌ 不支持 | 简单爬虫 |
| **DrissionPage** | 混合模式 | ~50MB | ✅ 完整 | ✅ 支持 | ✅ 兼容 | 国产推荐 |

## 二、Token 获取场景分析

### 为什么 Lightpanda 不适合 Token 获取？

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                         Token 获取流程                                       │
└─────────────────────────────────────────────────────────────────────────────┘
                                      │
                                      ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│                      Step 1: GitHub 登录                                     │
│                      需要用户输入账号密码                                      │
│                      ⚠️ Lightpanda: 无头模式，用户无法看到界面                │
└─────────────────────────────────────────────────────────────────────────────┘
                                      │
                                      ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│                      Step 2: 双因素认证 (可能)                                │
│                      需要用户输入验证码                                        │
│                      ⚠️ Lightpanda: 无法处理用户交互                          │
└─────────────────────────────────────────────────────────────────────────────┘
                                      │
                                      ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│                      Step 3: Token 创建页面                                   │
│                      复杂的 JavaScript 交互                                   │
│                      ⚠️ Lightpanda: Beta 阶段，可能不稳定                     │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 推荐方案

**Token 获取**：使用 Playwright (headless=False) 或手动方式
- 需要用户可见的浏览器界面
- 需要完整的 Web APIs 支持
- 需要稳定的 GitHub 兼容性

**其他自动化任务**：可考虑 Lightpanda
- 网页爬取
- 自动化测试
- 数据提取

## 四、未来可能的轻量化方案

### 方案 A: GitHub CLI (gh)
```bash
# 使用 GitHub 官方 CLI 工具
gh auth login
gh auth token
```
- 优点：官方工具，轻量
- 缺点：需要额外安装

### 方案 B: OAuth Device Flow
```
1. 脚本生成设备码
2. 用户在任意浏览器打开 GitHub 验证页面
3. 输入设备码完成授权
4. 脚本自动获取 Token
```
- 优点：不需要控制浏览器
- 缺点：需要 GitHub OAuth App

### 方案 C: SSH Key
```bash
# 使用 SSH 密钥代替 Token
ssh-keygen -t ed25519 -C "your_email@example.com"
# 将公钥添加到 GitHub
```
- 优点：更安全，无需 Token
- 缺点：配置稍复杂

## 五、结论

| 场景 | 推荐方案 |
|------|----------|
| **首次配置 Token** | Playwright (headless=False) 或手动获取 |
| **已有 Token** | 直接使用已保存的配置 |
| **CI/CD 环境** | 环境变量或 GitHub Actions secrets |
| **轻量化需求** | 考虑 GitHub CLI 或 SSH Key |

**Lightpanda 目前不适合 Token 获取场景**，但可以关注其发展，未来可能支持更多功能。
