## 二、功能模块

### 2.1 仓库配置模块

| 组件 | 功能 | 说明 |
|------|------|------|
| 仓库路径 | 选择本地 Git 仓库 | 支持浏览选择 |
| 远程地址 | 显示远程仓库 URL | 自动获取，可刷新 |
| 分支名称 | 选择目标分支 | 下拉框，支持自定义 |
| 提交信息 | 输入提交说明 | 可选，默认自动生成 |
| 强制推送 | 启用强制推送 | 会覆盖远程历史 |

### 2.2 Token 配置模块

| 组件 | 功能 | 说明 |
|------|------|------|
| Token | 输入 GitHub Token | 密码模式显示 |
| 获取按钮 | 获取新 Token | 支持手动/自动 |
| 显示按钮 | 切换 Token 可见性 | 安全考虑 |
| 用户名 | GitHub 用户名 | 可选 |
| 保存 Token | 保存到本地配置 | 下次自动加载 |
| 删除按钮 | 删除已保存的 Token | 清除配置 |

### 2.3 操作按钮

| 按钮 | 功能 | 说明 |
|------|------|------|
| 🚀 提交并推送 | 完整流程 | add + commit + push |
| 📝 仅提交 | 仅本地提交 | add + commit |
| 📤 仅推送 | 仅推送远程 | push |
| 🔄 检查状态 | 查看仓库状态 | git status |
| 🧹 清空日志 | 清空日志区域 | - |

## 六、使用方法

### 6.1 启动 GUI

```bash
python git_auto_push_gui.py
```

### 6.2 基本操作流程

1. **选择仓库**: 点击 [浏览] 选择本地 Git 仓库
2. **配置 Token**: 如果是私有仓库，点击 [获取] 获取 Token
3. **输入提交信息**: 填写本次提交的说明（可选）
4. **点击提交**: 点击 [🚀 提交并推送] 完成操作

### 6.3 快捷操作

- **仅提交**: 只在本地提交，不推送到远程
- **仅推送**: 只推送已有的提交到远程
- **检查状态**: 查看当前仓库的更改状态

## 七、技术实现

### 7.1 使用的库

| 库 | 用途 |
|------|------|
| tkinter | GUI 框架 |
| ttk | 主题化组件 |
| threading | 后台线程执行 |
| filedialog | 文件选择对话框 |
| messagebox | 消息对话框 |
| scrolledtext | 滚动文本框 |

### 7.2 线程安全

- Git 操作在后台线程中执行，避免界面卡顿
- 使用 `root.after()` 安全更新 GUI 组件
- 日志输出实时显示，不阻塞主线程

### 7.3 日志颜色

| 颜色 | 含义 |
|------|------|
| 🟢 绿色 | 成功 |
| 🔴 红色 | 错误 |
| 🟠 橙色 | 警告 |
| 🔵 蓝色 | 信息 |
